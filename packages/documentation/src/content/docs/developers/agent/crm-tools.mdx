The agent ships with a set of CRM tools that let it query contacts, deals, call history, and integrations directly from your workspace. These are created via `createCrmTools()` and registered into the tool registry.

## Available Tools

| Tool | Description |
| --- | --- |
| `search_contacts` | Search contacts by name, email, phone, or company |
| `get_contact` | Fetch a single contact by ID with full details |
| `search_deals` | Query deals by stage, amount, or close date |
| `get_deal` | Fetch a single deal with related contacts and activities |
| `list_calls` | List recent call records with duration and outcome |
| `list_integrations` | Show connected integrations and their status |

## Context Loading

Before each chat turn, the agent loads workspace context via the `ContextLoader` interface:

```typescript
type ContextLoader = {
  load: (userId: string, workspaceId: string) => Promise<AgentContext>;
};
```

The loaded context includes:

- **Active call** — If the user is on a call, the agent sees the contact name, direction, and call SID
- **Recent activity** — Last 10 CRM activities (calls, emails, notes, meetings)
- **Connected integrations** — Which third-party services are linked
- **Memories** — Up to 5 stored agent memories for personalization

## System Prompt Injection Defense

CRM data is semi-trusted — it comes from user input stored in the database. The agent wraps CRM context in XML delimiters to prevent prompt injection:

```
<crm_context>
Active call: outbound with Jane Smith (ID: abc123)
Recent activity:
- [call] Discussed renewal options
- [email] Sent follow-up proposal
</crm_context>
```

Field values are sanitized before injection:
- Newlines and carriage returns replaced with spaces
- Backtick fences stripped to prevent code block escapes

<Note>
  The agent also instructs the LLM to never send data to URLs outside connected
  integrations, providing an additional layer of data exfiltration defense.
</Note>

## CRM Client

The `CrmClient` class handles GraphQL communication with the CRM backend:

```typescript
import { CrmClient } from '@consuelo/agent';

const crm = new CrmClient({
  graphqlUrl: 'https://api.consuelohq.com/graphql',
  token: '<jwt-access-token>',
});

const contacts = await crm.searchContacts({
  query: 'Jane',
  filters: { company: 'Acme' },
  pagination: { limit: 10, offset: 0 },
});
```

The client uses the same JWT authentication as the rest of the Consuelo API. See the [Authentication Guide](/developers/api/auth) for token details.
