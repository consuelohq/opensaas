The agent package includes an integration framework for connecting third-party services and an observability layer powered by Langfuse.

## Integration Registry

Integrations are defined in a central registry with their auth method, capabilities, and configuration:

```typescript
import { INTEGRATION_REGISTRY, IntegrationConnectionService } from '@consuelo/agent';

// list all available integrations
const integrations = INTEGRATION_REGISTRY;

// manage connections for a user
const service = new IntegrationConnectionService({
  store,        // IntegrationStore implementation
  encrypt,      // EncryptFn for token storage
  decrypt,      // DecryptFn for token retrieval
});

await service.connect(userId, 'ghl', { apiKey: '<key>' });
```

Each integration defines:

- **Category** — `crm`, `telephony`, `email`, `calendar`, `analytics`
- **Auth method** — `oauth2`, `api_key`, or `webhook`
- **Capabilities** — What the integration can do (sync contacts, push calls, etc.)

### Sandbox Environment

When the agent runs code in the sandbox, connected integrations can inject environment variables:

```typescript
import { buildSandboxEnv } from '@consuelo/agent';

const env = await buildSandboxEnv(userId, connections);
// { GHL_API_KEY: '...', TWILIO_ACCOUNT_SID: '...' }
```

## Tracing with Langfuse

The `TracingService` provides optional observability via [Langfuse](https://langfuse.com). It tracks every LLM call, tool invocation, and token usage.

```typescript
import { TracingService } from '@consuelo/agent';

const tracing = new TracingService({
  langfuse: {
    publicKey: process.env.LANGFUSE_PUBLIC_KEY,
    secretKey: process.env.LANGFUSE_SECRET_KEY,
    baseUrl: 'https://cloud.langfuse.com',  // optional
  },
  executionStore,  // optional — persists execution history to DB
});
```

### What Gets Traced

| Event | Data Captured |
| --- | --- |
| **Trace start** | User ID, conversation ID, skill ID |
| **LLM generation** | Model name, input/output token counts |
| **Tool call** | Tool name, arguments, result |
| **Flush** | Sends buffered events to Langfuse |

Tracing is non-fatal — if Langfuse is unavailable or misconfigured, the agent continues working normally.

<Note>
  Langfuse is a peer dependency. Install it separately with
  `yarn add langfuse` if you want tracing enabled.
</Note>

## Streaming Protocol

The agent uses the AI SDK UI message stream protocol for real-time responses. This handles text chunks, tool calls, and tool results in a single stream.

```typescript
import { handleChat } from '@consuelo/agent';

const { conversationId, response } = await handleChat(
  { message: 'What deals are closing this week?' },
  userId,
  workspaceId,
  { config, store, contextLoader, tools, tracing },
);

// response is a standard Response object with a ReadableStream body
// compatible with any framework (Express, Hono, Next.js, etc.)
return response;
```

The frontend consumes this with the AI SDK `useChat` hook or any client that speaks the UI message stream protocol.

### Conversation Persistence

Each chat turn automatically:

1. Loads or creates a conversation from the `ConversationStore`
2. Appends the user message
3. Streams the assistant response
4. Persists the final text and token usage after the stream completes
