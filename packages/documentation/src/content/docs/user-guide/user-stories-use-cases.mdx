# User Stories & Use Cases

Consuelo OS is for teams that want agents to do useful work across real tools, real data, and human approval steps.

This page is a structure-first draft. The goal is to make the docs easier to navigate before every story is fully written.

## Overview

Consuelo OS is an open-source gateway that connects agents to company tools, workflows, files, approvals, and human-facing products.

A team can use Consuelo OS directly as an agent workspace, or through tool packs such as Sites, Office, Media, Browser, Workspace, and GraphQL-backed data tools.

## Sales Teams

Sales teams use Consuelo OS to manage records, follow up with prospects, run workflows, and keep human sellers in control while agents handle repetitive work.

Good starting points:

- Sites for GTM workflows and customer-facing pages
- Dialer and messaging workflows
- Data model and pipeline views
- GraphQL-backed automations

## Insurance Agencies

Insurance agencies use Consuelo OS to organize contacts, policies, conversations, follow-ups, and operational workflows in one place.

Good starting points:

- Contact and account data
- Dialer workflows
- Files and artifacts
- Human approval flows

## Founder / Operator Workflows

Founders and operators use Consuelo OS as a batteries-included operating layer for tasks that cross tools.

Good starting points:

- Workspace tools
- Browser workflows
- Office artifacts
- Sites for lightweight GTM surfaces

## Support & Ops

Support and operations teams use Consuelo OS to coordinate work, route requests, keep context attached to records, and make sure agents ask for approval when work needs a human decision.

Good starting points:

- Workflows
- Files and artifacts
- Approvals
- Messaging and collaboration tools

## Developer / Automation Workflows

Developers and automation teams use Consuelo OS to add tools, expose data safely, and build repeatable workflows that agents can call.

Good starting points:

- Tool contracts
- GraphQL API
- OS agent context
- Skills and scripts
