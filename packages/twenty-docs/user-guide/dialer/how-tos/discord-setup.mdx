> Get the Consuelo bot running in your Discord server.

This guide walks you through setting up the Consuelo Discord bot for your workspace.

## Prerequisites

Before starting, you'll need:

- A Consuelo workspace with admin access
- Discord server where you can add bots
- Your Consuelo API key (found in Settings → API)
- Your workspace ID and user ID

## Step 1: Invite the Bot

### For Cloud-hosted Consuelo

The bot is hosted by Consuelo. Just invite it to your server:

1. Visit the invite link (provided by your Consuelo admin)
2. Select your Discord server
3. Grant the requested permissions
4. Authorize the bot

### For Self-Hosted

If you're self-hosting Consuelo, deploy the bot separately:

```bash
cd packages/chat-bot
npm install
npm run build
npm start
```

Set these environment variables:

```bash
DISCORD_BOT_TOKEN=your_discord_bot_token
CONSUELO_API_URL=https://your-consuelo-instance.com
CONSUELO_API_KEY=your_api_key
REDIS_URL=redis://localhost:6379
PORT=3100
```

## Step 2: Link Your Account

Each user must link their Discord account to their Consuelo workspace.

### Find Your Credentials

1. **Workspace ID**: Go to Settings → General, copy the workspace ID
2. **User ID**: Go to Settings → Profile, your user ID is shown there
3. **API Key**: Go to Settings → API, generate a new API key

### Run the Link Command

```
/consuelo login
```

The bot will respond with a link:

```
🔗 Link Your Account

Click below to connect your Discord account to your Consuelo workspace.

[Link Account]
```

Click the link and enter:

- **Workspace ID**: Your workspace identifier
- **User ID**: Your user identifier
- **API Key**: Your generated API key

Click **Link Account** — you'll see a success message.

### Verify the Link

```
/consuelo ping
```

You should see:

```
🏓 Pong!

Version: 0.0.1
Uptime: 2h 15m
Workspace: your-workspace-id
Status: Online
```

## Step 3: Set Your Phone Number

The bot needs to know which phone number to call when connecting you to leads.

### Via CRM

1. Log into Consuelo CRM
2. Go to **Settings** → **Profile**
3. Add your phone number in the **Phone** field
4. Save changes

### Via API (Advanced)

```bash
curl -X PATCH https://api.consuelohq.com/v1/users/me \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"phone": "+15551234567"}'
```

## Step 4: Configure Notifications (Optional)

Set a channel for automatic call notifications.

### Requirements

- You must be a workspace admin
- The bot must have permission to post in the channel

### Command

```
/consuelo config channel #call-notifications
```

Replace `#call-notifications` with your actual channel name.

**Success response:**

```
✅ Channel Set

Notifications will be posted to #call-notifications.
```

## Step 5: Test Your Setup

### Quick Health Check

```
/consuelo status
```

Should show all services as ✅

### Test Contact Search

```
/consuelo contacts search test
```

Should return contacts or "No Results" (not an error).

### Test Queue Access

```
/consuelo queue status
```

Should show your queues or "No queues found."

## Troubleshooting

### "Not Linked" Error

**Problem**: You haven't linked your Discord account.

**Solution**: Run `/consuelo login` and complete the link process.

### "Missing phone number" Error

**Problem**: Your phone number isn't set in Consuelo.

**Solution**: Add your phone in CRM Settings → Profile.

### "Unauthorized" Error

**Problem**: Your API key is invalid or expired.

**Solution**:

1. Go to CRM Settings → API
2. Revoke the old key
3. Generate a new key
4. Run `/consuelo logout` then `/consuelo login` again

### Bot Not Responding

**Problem**: Bot is offline or doesn't have permissions.

**Solution**:

1. Check the bot's status in your Discord server
2. Verify the bot has permission to read messages and send messages
3. For self-hosted: Check that the bot service is running

### Can't Set Notification Channel

**Problem**: Missing admin permissions or bot permissions.

**Solution**:

1. Verify you're a workspace admin in Consuelo
2. Check that the bot has "Send Messages" permission in the channel
3. Try using the channel ID instead of the mention

## Security Notes

<Warning>
  **Protect your API key!** - Never share your API key in public channels - Use
  DMs for the `/consuelo login` link - Regenerate keys if you suspect compromise
  - The bot stores credentials encrypted in Redis
</Warning>
