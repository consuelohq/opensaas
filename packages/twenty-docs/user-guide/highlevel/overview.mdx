> Use Consuelo in HighLevel two ways: run the embedded dialer inside HighLevel, or connect your HighLevel location to sync contacts and pipelines with your CRM.

Consuelo supports two separate HighLevel experiences. They solve different problems and they can be used together.

## Choose the right setup

| Option          | Best for                         | Lives in          | What it does                                                                            |
| --------------- | -------------------------------- | ----------------- | --------------------------------------------------------------------------------------- |
| Embedded Dialer | Reps who work inside HighLevel all day | HighLevel sidebar       | Opens Consuelo in an iframe, handles click-to-call, secure sign-in, and in-call work    |
| Integration     | Admins connecting systems        | Consuelo settings | Connects a HighLevel location over OAuth, imports contacts, syncs updates, and maps pipelines |

## When to use each mode

### Embedded dialer

Choose the embedded dialer when your team needs to stay inside HighLevel while calling. The widget loads at calls.consuelohq.com/calls/embedded, validates that it is running inside an iframe, and keeps the current HighLevel contact in context for click-to-call, queue actions, and call history.

### Integration

Choose the integration when you want data to move between HighLevel and Consuelo. The integration uses an OAuth connection, stores encrypted access tokens, imports HighLevel contacts into People records, receives webhook events, and lets you map HighLevel pipeline stages to your pipeline stages.

## Use both together

Many teams use both:

- reps launch the embedded dialer from the HighLevel sidebar
- admins connect the integration so contacts, pipelines, and logged outcomes stay aligned

<Note>
  The embedded dialer and the data integration are related, but they are not the
  same product surface. The embedded experience runs in HighLevel. The
  integration settings live in the main Consuelo app.
</Note>
