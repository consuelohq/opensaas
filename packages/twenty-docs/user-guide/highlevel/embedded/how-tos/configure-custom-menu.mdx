title: Configure the Custom Menu
description: Point the HighLevel sidebar entry at the embedded Consuelo dialer.
---

The embedded dialer works best when the HighLevel sidebar opens the correct widget URL every time.

## Menu target

Use this URL for the embedded dialer entry:

https://calls.consuelohq.com/calls/embedded

## Recommended menu behavior

Configure the HighLevel sidebar entry so reps can open the dialer from the left navigation without leaving the CRM.

Recommended setup:

- clear dialer label in the sidebar
- open the widget inside the embedded HighLevel app surface
- avoid sending users to a standalone browser tab

## Why this matters

The embedded app expects all of these things to be true:

- it is running inside an iframe
- the parent frame is a trusted HighLevel origin
- the dialer can exchange sign-in and click-to-call messages with that parent frame

If the sidebar points anywhere else, the embedded checks and messaging flow can fail.

## After updating the menu

Reload HighLevel and confirm:

- the dialer opens inside the sidebar
- the widget no longer shows the embedded-only warning
- click-to-call works from a contact context
