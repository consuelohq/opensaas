> The data model defines the business objects agents can operate.

## Decision

The data model is the OS ontology. It is how the customer business becomes machine-operable.

## Core objects

- People
- Companies
- Lists
- Calls
- Files
- Tasks
- Notes
- Workflows
- Workflow Runs
- Dashboards

## Revenue/insurance extensions

- Campaigns
- Landing Pages
- Ads
- Scripts
- Objections
- Appointments
- Lead Sources
- Reports

## Rule

Runbooks should access business data through facades so the underlying model can evolve without rewriting every runbook.
