> GraphQL facades centralize backend data access for runbooks.

## Purpose

GraphQL facades give runbooks a stable internal API for workspace data.

## Rule

Runbooks should not scatter raw GraphQL queries everywhere.

Use facades for workspace context, people, companies, lists, calls, tasks, files, workflows, dashboards, and analytics.

## Early proof

The first smoke runbook can use internal GraphQL credentials through env to prove connectivity.

Required env names from the OS scaffold:

- `CONSUELO_GRAPHQL_URL`
- `CONSUELO_INTERNAL_GRAPHQL_API_KEY`

## Secret safety

GraphQL API keys must stay in env. They should never appear in code, docs, logs, fixtures, committed config, or rendered output.
