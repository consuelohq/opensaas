> Business and revenue steering for agents operating a Consuelo workspace.

## Purpose

`get_steering` gives a revenue/business agent enough context to start useful work without rediscovering the workspace.

The response can be large. Useful steering is more important than a tiny response that forces repeated lookup.

## Should include

- OS identity
- workspace identity
- user role when available
- rules
- permissions
- business context
- data model overview
- installed runbooks
- raw runbook manifest
- integration status
- current workspace state when available
- docs links
- example `call` invocations

## Implementation notes

Secrets must never appear in steering. The raw manifest should be included so agents know exact available runbooks.
