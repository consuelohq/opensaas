> The single execution gateway for Consuelo OS runbooks.

## Purpose

`call` executes a named runbook through the OS runtime.

MCP tools are entrypoints. Runbooks are capabilities. Package scripts power the OS.

## Input shape

```json
{"name":"daily-revenue-brief","input":{},"workspaceId":"optional","userId":"optional"}
```

## Responsibilities

`call` should resolve the runbook, validate input, check permission metadata, route to Bun, call GraphQL facades, use filesystem/artifacts, use sandbox execution when needed, return structured output, log enough metadata to debug, and avoid leaking secrets.

## Constraint

Do not expose each runbook as a separate MCP tool. Keep the public agent interface stable and route business capabilities through `call`.
