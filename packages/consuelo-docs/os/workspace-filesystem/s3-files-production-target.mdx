> S3 Files is the preferred production filesystem target for Consuelo OS.

## Decision

S3 Files-mounted workspace is the preferred production target for Consuelo OS filesystem-heavy execution.

## Why

Consuelo OS needs Linux compute, normal CLI/file operations, shared workspace paths, durable artifacts, and runbook outputs.

## Production shape

```text
Linux executor
  + mounted workspace filesystem
  + Bun runtime
  + GraphQL/API credentials
  + sandbox/executor
  + runbook registry
```

## Compatibility

Current S3 object storage remains useful as fallback/compatibility. The OS file API should expose workspace paths, not raw object keys.

## Open question

Runtime hosting must confirm where S3 Files can be mounted. If Railway cannot mount the production filesystem directly, filesystem-heavy executor jobs may need AWS compute while app/API can remain elsewhere.
