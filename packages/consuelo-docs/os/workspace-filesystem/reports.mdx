> Reports are structured artifacts for managers, reps, and operators.

## Definition

A report is a human-readable artifact generated from workspace data and runbook logic.

## Examples

- daily revenue brief
- weekly manager report
- call coaching report
- campaign performance report
- lead priority report

## Target behavior

Reports should be saved under `/workspace/reports`. Runbook-specific intermediate output should be saved under `/workspace/runbook-runs`.
