> Local filesystem behavior for development and smoke testing.

## Purpose

Local filesystem support lets the OS runtime and skills work before production filesystem infrastructure is finalized.

## Target behavior

- skills can read and write workspace paths locally
- local outputs follow the canonical workspace layout
- local artifacts have the same metadata shape as production artifacts
- local paths remain behind an abstraction

## Constraint

Local development paths should stay behind the filesystem abstraction. Production target is a mounted workspace filesystem, preferably S3 Files.
