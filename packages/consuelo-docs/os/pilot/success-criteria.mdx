> Measurable criteria for the first Consuelo OS pilot.

## Product success

- agent can orient itself with steering
- agent can use real workspace data
- agent can run daily revenue brief
- agent can create useful artifacts
- agent can propose writes safely
- permissions are enforced
- internal team can debug runs
- customer can repeat workflow from docs

## Business success

- reps get clearer next actions
- manager gets useful reports
- follow-ups improve
- call insights become actionable
- landing page/ad feedback uses real revenue context
- onboarding gets easier

## Technical success

- MCP surface stays small
- runbooks route through `call`
- manifest stays source of truth
- GraphQL facades centralize data access
- filesystem abstraction supports runbook artifacts
- logs and errors are inspectable
