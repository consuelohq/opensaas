> Permission levels and approval behavior for Consuelo OS skills.

## Permission levels

- `read`: read workspace data and return analysis.
- `draft`: create drafts, files, reports, briefs, scripts, or proposed changes.
- `write`: create or update internal records.
- `execute`: run sandboxed code, scripts, analysis, or workflow logic.
- `external`: trigger external side effects such as messages, calls, publishing, or ad changes.
- `admin`: change configuration, users, secrets, schemas, integrations, billing, or workspace settings.

## Approval rules

Read can usually run within user scope. Draft can usually create artifacts. Write requires role permission and may require approval. Execute requires runtime permission. External requires explicit approval. Admin is owner/operator scoped.
