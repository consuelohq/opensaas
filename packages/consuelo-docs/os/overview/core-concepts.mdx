> Core terms and operating concepts for Consuelo OS.

## OS

Managed operating layer agents use to work inside a revenue business.

## Workspace

Customer-scoped environment containing users, data, files, integrations, skills, permissions, artifacts, and logs.

## Data Model

The OS ontology: business objects agents can reason about and act on.

## Skill

Documented business capability behind `call`.

## Tool Manifest

Skill registry with capability metadata, permissions, inputs, outputs, env, and integrations.

## Steering

Context an agent reads before acting. `get_steering` is business steering; `get_dev_steering` is build/design/operator steering.

## Artifact

Durable skill output such as a report, brief, draft, call summary, script, or run output.

## Sandbox

Cloud computer/executor for isolated work.

## Agent

ChatGPT, Claude, local coding agents, or other model-backed workers connected to Consuelo OS.

## Operator

Internal Consuelo role or admin script for provisioning, debugging, configuration, and workspace health.

## Permission Level

What a skill can do: `read`, `draft`, `write`, `execute`, `external`, or `admin`.

## Filesystem

Workspace path surface for skills and agents; local in development, S3 Files-mounted in production.

## GraphQL Facade

Stable data access layer so skills avoid scattered raw GraphQL queries.

## Decision Engine

Combines structured facts, vectorized context, business rules, skill logic, and outcomes to recommend next action.
