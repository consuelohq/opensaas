> The OS portal is small while the package/operator surface stays powerful.

## Core rule

OS portal entrypoints are entrypoints. Skills are capabilities. Package scripts power the OS.

## OS portal entrypoints

- `get_steering`
- `get_dev_steering`
- `call`

## Package scripts

Package scripts are internal/runtime/operator commands. They can be numerous because they power the OS behind the small OS portal.

## Important distinction

Keeping three OS portal entrypoints does not mean keeping three package scripts. The OS needs a powerful internal surface and a simple agent entrypoint.
