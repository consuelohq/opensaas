> Scheduled runbooks are recurring OS jobs such as daily briefs.

## Purpose

Scheduled runbooks let the OS run recurring workflows without a human prompt.

## First candidates

- daily revenue brief
- weekly manager report
- missed follow-up report
- campaign performance brief
- stale lead detection

## Existing primitive

Consuelo has worker/cron potential. V1 should document and scaffold scheduled runbook concepts before overbuilding a scheduler.
