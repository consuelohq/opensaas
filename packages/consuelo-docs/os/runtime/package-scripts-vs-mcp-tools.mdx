> The MCP surface is small while the package/operator surface stays powerful.

## Core rule

MCP tools are entrypoints. Runbooks are capabilities. Package scripts power the OS.

## MCP tools

- `get_steering`
- `get_dev_steering`
- `call`

## Package scripts

Package scripts are internal/runtime/operator commands. They can be numerous because they power the OS behind the small MCP surface.

## Important distinction

Keeping three MCP tools does not mean keeping three package scripts. The OS needs a powerful internal surface and a simple agent entrypoint.
