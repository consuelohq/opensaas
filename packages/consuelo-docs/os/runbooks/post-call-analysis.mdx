> Reviews a call and extracts outcome, objections, coaching notes, and follow-up recommendations.

## Purpose

Reviews a call and extracts outcome, objections, coaching notes, and follow-up recommendations.

## Status

Scaffolded unless the manifest or implementation says otherwise.

## Permission level

`draft`

## Inputs

call record, transcript, person/company record, script, outcome.

## Outputs

summary, objections, sentiment, next action, proposed notes.

## Artifacts

This runbook should return artifact references when it creates durable files, reports, briefs, scripts, or outputs.

## Approval behavior

Read and draft work can run within user scope. Record writes require write permission. External actions require explicit approval.

## Implementation notes

Can draft notes and tasks. Record writes should remain gated.

## Failure modes

- missing env
- missing integration credentials
- incomplete data model fields
- no relevant workspace data
- sandbox unavailable when execution is required

The runbook should return structured failure status instead of crashing the OS flow.
