## Overview

Computer isn't just a conversational assistant — it can actually do things in your CRM. These are the tools at its disposal.

## Available Tools

### Search Contacts

Find contacts by name, email, phone, or company.

| Parameter | Type   | Description                       |
| --------- | ------ | --------------------------------- |
| query     | string | Search query (name, email, phone) |
| limit     | number | Max results (default 50)          |

**Example:**

> "Find contacts at Acme Corp"
> "Search for john@example.com"
> "Who do I know at Salesforce?"

**Returns:** List of matching contacts with IDs, names, companies, and contact info.

---

### Get Contact Details

Retrieve full contact record including notes and activity.

| Parameter | Type   | Description |
| --------- | ------ | ----------- |
| contactId | string | Contact ID  |

**Example:**

> "Show me John Smith's full profile"
> "What's the history with Acme Corp?"
> "Get details for this contact"

**Returns:** Complete contact record with all fields and recent activity.

---

### List Deals

View opportunities filtered by pipeline stage.

| Parameter | Type   | Description                         |
| --------- | ------ | ----------------------------------- |
| stage     | string | Filter by pipeline stage (optional) |
| limit     | number | Max results (optional)              |

**Example:**

> "What deals are closing this month?"
> "Show me all negotiations"
> "List opportunities over $50,000"

**Returns:** List of deals with stage, amount, close date, and contact.

---

### Get Call History

View recent calls with filters.

| Parameter | Type   | Description                        |
| --------- | ------ | ---------------------------------- |
| contactId | string | Filter by contact (optional)       |
| since     | string | ISO date to filter from (optional) |
| limit     | number | Max results (optional)             |

**Example:**

> "What calls did I have with Acme this week?"
> "Show my call history for the last 7 days"
> "Get all calls to John Smith"

**Returns:** List of call records with timestamps, durations, and outcomes.

---

### Search Knowledge Base

Search uploaded documents and resources.

| Parameter  | Type   | Description                             |
| ---------- | ------ | --------------------------------------- |
| query      | string | Search query                            |
| collection | string | Limit to specific collection (optional) |

**Example:**

> "Search knowledge base for pricing"
> "What does our product doc say about integrations?"
> "Find information about enterprise features"

**Returns:** Relevant document sections with source citations.

---

### Log Call

Record a call outcome in the CRM.

| Parameter | Type   | Description                                  |
| --------- | ------ | -------------------------------------------- |
| contactId | string | Contact ID                                   |
| outcome   | string | One of: answered, voicemail, no_answer, busy |
| notes     | string | Call summary                                 |
| nextStep  | string | Follow-up action (optional)                  |

**Example:**

> "Log a call with John Smith — he's interested in enterprise, follow up next week"
> "Record that I left a voicemail for Acme Corp"

**Returns:** Created call record with ID and timestamp.

---

### Update Deal

Modify opportunity stage, amount, or notes.

| Parameter | Type   | Description                   |
| --------- | ------ | ----------------------------- |
| id        | string | Deal ID                       |
| stage     | string | New pipeline stage (optional) |
| amount    | number | New deal amount (optional)    |
| notes     | string | Notes to append (optional)    |

**Example:**

> "Move Acme Corp to Negotiation stage"
> "Update the deal amount to $50,000"
> "Add a note about the pricing discussion"

**Returns:** Updated deal record.

<Warning>
  Stage changes require explicit evidence. Computer won't advance deals on vague
  commitments.
</Warning>

---

### Create Note

Add a note to a contact record.

| Parameter | Type   | Description  |
| --------- | ------ | ------------ |
| contactId | string | Contact ID   |
| content   | string | Note content |

**Example:**

> "Add a note to John Smith: interested in enterprise plan"
> "Note that Acme Corp mentioned a Q4 budget discussion"

**Returns:** Created note with ID and timestamp.

---

### Create Task

Schedule a follow-up task.

| Parameter | Type   | Description                   |
| --------- | ------ | ----------------------------- |
| title     | string | Task title                    |
| dueDate   | string | ISO date string               |
| contactId | string | Associated contact (optional) |

**Example:**

> "Create a follow-up task for Friday — send proposal to Acme"
> "Remind me to call John Smith next Tuesday"

**Returns:** Created task with ID and due date.

---

### Add to Queue

Queue contacts for calling.

| Parameter  | Type     | Description                         |
| ---------- | -------- | ----------------------------------- |
| contactIds | string[] | List of contact IDs                 |
| priority   | number   | Queue priority 1=highest (optional) |

**Example:**

> "Add these 5 contacts to my call queue"
> "Queue Acme Corp as priority 1"

**Returns:** Queue position and estimated wait time.

---

### Get Analytics

Retrieve call metrics and performance data.

| Parameter | Type   | Description                               |
| --------- | ------ | ----------------------------------------- |
| type      | string | Metric type: calls, answer_rate, duration |
| period    | string | Time period: today, week, month           |
| groupBy   | string | Group by: day, hour, agent (optional)     |

**Example:**

> "What's my answer rate this week?"
> "Show call volume by day for the month"
> "How does my talk time compare to last month?"

**Returns:** Aggregated metrics with trend data.

## How Computer Chooses Tools

Computer doesn't just execute tools randomly — it reasons about what's needed:

1. **Understand the request** — parse natural language into intent
2. **Identify required tools** — which tools can satisfy the request
3. **Gather parameters** — ask clarifying questions if needed
4. **Execute** — call the tool with correct parameters
5. **Present results** — summarize the output in natural language

### Example Flow

**User:** "What opportunities are closing this month?"

**Computer's process:**

1. Intent: List deals filtered by close date
2. Tool: `list_deals` with stage filter or date range
3. Execute: Query CRM
4. Result: 5 deals closing in March
5. Present: "You have 5 deals closing this month totaling $127,000..."

## Permissions

Each tool respects role-based access control:

| If your role can't... | Then Computer can't...      |
| --------------------- | --------------------------- |
| View contacts         | Search or get contacts      |
| Edit deals            | Update deal stage or amount |
| Create tasks          | Schedule follow-ups         |
| View analytics        | Get performance metrics     |

Configure permissions in **Settings → Roles**.
