## Overview

Computer doesn't work in isolation — it understands where you are and what you're working on. This context awareness means suggestions are always relevant to the current situation.

## Types of Context

### Contact Context

When you're viewing a contact, Computer knows:

| Data         | Description                              |
| ------------ | ---------------------------------------- |
| Basic info   | Name, title, company, email, phone       |
| Company      | Industry, size, location, website        |
| Deal history | All opportunities linked to this contact |
| Activity     | Calls, emails, meetings, notes           |
| Pipeline     | Where their deal stands                  |

**Example:**

When coaching during a call with John Smith from Acme Corp, Computer shows:

```
Context for John Smith:
├─ Company: Acme Corp (Enterprise, 500+ employees)
├─ Deal: $24,000/year, Stage: Negotiation
├─ Title: VP of Engineering
├─ Last contact: Called 3 days ago
├─ Key notes:
│   • Interested in enterprise features
│   • Price objection: "Seems expensive"
│   • Competitor: Using [Legacy Tool]
│   • Decision maker: Needs CFO approval
└─ Next Step: Send case study, schedule demo
```

### Deal Context

When viewing an opportunity, Computer knows:

| Data        | Description                    |
| ----------- | ------------------------------ |
| Stage       | Pipeline position              |
| Amount      | Deal value                     |
| Probability | Win likelihood                 |
| Close date  | Expected close                 |
| Contacts    | All people linked to this deal |
| Activity    | Timeline of interactions       |
| Stalled?    | How long in current stage      |

**Example:**

When discussing this deal, Computer suggests:

> "This deal has been in Negotiation for 22 days — average is 8 days. Consider asking: 'What's blocking your decision?' or 'Who else needs to approve this?'"

### Pipeline Context

Computer understands your entire pipeline:

- Total pipeline value
- Deals by stage
- Stuck deals (longer than average)
- Closing this month
- At-risk deals

**Example:**

> "You have $127K closing this month across 5 deals. Two deals in Negotiation are past the average time — Acme Corp and TechStart. Would you like suggested outreach for each?"

## How Context is Injected

### Context Injection Pipeline

When Computer activates, it follows this pipeline:

```
1. Call Detection
   │
   ├─ Recognize call is active
   ├─ Identify contact (via phone number)
   └─ Load contact record
       │
2. Data Enrichment
   │
   ├─ Fetch company details
   ├─ Fetch deal history
   ├─ Fetch recent activity
   └─ Fetch pipeline position
       │
3. Skill Selection
   │
   ├─ Determine current mode (coaching vs. analysis)
   ├─ Load appropriate skill
   └─ Inject context into skill prompt
       │
4. Ongoing Updates
   │
   ├─ Transcribe conversation
   ├─ Update context based on new info
   └─ Adjust suggestions in real-time
```

### Before-Turn Extensions

Computer uses extensions that inject context before each AI turn:

| Extension            | What It Does                       |
| -------------------- | ---------------------------------- |
| ContextInjection     | Loads contact, deal, company data  |
| PipelineIntelligence | Analyzes pipeline stage and timing |
| CoachingDetector     | Detects when coaching is needed    |
| CoachingLifecycle    | Manages coaching session flow      |

## Context in Action

### During a Call

**Before coaching activates:**

Computer pre-loads:

```
Customer: John Smith at Acme Corp
├─ Deal: $24K/year, Negotiation stage, 14 days in stage
├─ Previous calls: 3 calls in last month
├─ Last outcome: "Interested, needs CFO approval"
├─ Competitor: Currently using [Legacy Tool]
└─ Pain point: Legacy Tool doesn't integrate with Salesforce
```

**During coaching:**

As the conversation progresses, the context updates in real-time:

```
Update (2:30): Customer mentioned budget approval in Q2
→ Suggest: "So Q2 budget approval — that's April to June, right?
  Would a June implementation timeline work for your team?"

Update (5:15): Customer asked about implementation time
→ Suggest: "Implementation typically takes 2-3 weeks.
  When would you want to start if you move forward?"
```

### After a Call

Context helps shape accurate analysis:

```
Analysis based on loaded context:
├─ Deal stage: Negotiation
├─ Days in stage: 14 (above average of 8)
├─ Commitment: Demo scheduled for Friday ✓
├— Recommendation: Move to "Demo Scheduled" stage
│
└─ Evidence for stage change:
    Customer: "Let's do the demo on Friday"
    Rep: "Perfect, I'll send the invite"
    Customer: "I'll make sure my CFO joins"
```

## Context Sources

Computer pulls context from multiple sources:

| Source         | Data Type                           |
| -------------- | ----------------------------------- |
| CRM Database   | Contacts, companies, deals          |
| Activity Feed  | Calls, emails, meetings, notes      |
| Knowledge Base | Uploaded documents, resources       |
| Integrations   | Connected tools (GoHighLevel, etc.) |
| Real-Time      | Current conversation transcript     |

## Tips for Better Context

### Keep Records Updated

Computer's suggestions are only as good as the data:

- Log calls consistently
- Update deal stages accurately
- Add notes after important conversations
- Keep contact info current

### Use the Knowledge Base

Upload frequently referenced content:

- Product documentation
- Pricing sheets
- Case studies
- Competitor comparisons

Computer searches this during calls:

> Customer: "How does this compare to [Competitor]?"
>
> Computer searches KB → shows relevant comparison from uploaded docs

### Tag Contacts Correctly

Proper tags help Computer understand context faster:

- Decision Maker
- Budget Authority
- Technical Evaluator
- Champion
- Influencer
