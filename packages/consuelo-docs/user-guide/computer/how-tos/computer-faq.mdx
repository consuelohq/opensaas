## General

<AccordionGroup>

<Accordion title="What is Computer?">

Computer is Consuelo's AI assistant for sales teams. It provides real-time coaching during calls, analyzes conversations after calls, and automates CRM tasks like logging calls, updating deals, and creating follow-up tasks.

Unlike generic AI chatbots, Computer has deep integration with your CRM data — it knows your contacts, deals, and pipeline, and can take actions directly in your CRM.

</Accordion>

<Accordion title="Is Computer available now?">

Yes. Computer is available in Consuelo. The real-time coaching requires the dialer to be active. CRM actions and context awareness are available when viewing contacts, companies, and opportunities.

</Accordion>

<Accordion title="What's the difference between Computer and the AI Chatbot in Consuelo?">

Consuelo's Computer is a focused sales assistant integrated with the dialer for real-time coaching. It's designed specifically for sales teams making calls:

- Activates automatically during calls
- Provides live coaching based on conversation
- Logs calls and updates deals automatically
- Understands pipeline and deal context

The generic AI chatbot in Consuelo CRM is for general queries, while Computer is purpose-built for sales workflows.

</Accordion>

<Accordion title="Do I need special permissions to use Computer?">

Computer respects your existing role permissions. If your role allows you to view contacts and update deals, Computer can do the same. Administrators can configure Computer's access separately in **Settings → Roles**.

</Accordion>

</AccordionGroup>

## Real-Time Coaching

<AccordionGroup>

<Accordion title="When does coaching activate?">

Coaching activates automatically when:

- You make an outbound call through the dialer
- You answer an inbound call
- You start a call from a contact page (click-to-call)

The coaching panel opens automatically and loads context for the contact you're speaking with.

</Accordion>

<Accordion title="Can I turn off coaching?">

Yes. During a call, you can:

- Click the X on the coaching panel to minimize it
- Use the keyboard shortcut to toggle coaching visibility
- Disable coaching entirely in **Settings → Preferences**

</Accordion>

<Accordion title="How accurate is the transcription?">

Transcription accuracy depends on audio quality. For best results:

- Use a quality headset
- Minimize background noise
- Speak clearly
- Ensure stable internet connection

Computer uses real-time speech recognition optimized for sales conversations.

</Accordion>

<Accordion title="Can Computer understand my customer?">

Computer transcribes both sides of the conversation using speaker diarization — it can distinguish between you and the customer. This enables:

- Objection detection from customer statements
- Question analysis
- Buying signal recognition
- Accurate sentiment tracking

</Accordion>

</AccordionGroup>

## Post-Call Analysis

<AccordionGroup>

<Accordion title="What happens after a call?">

After every call, Computer automatically:

1. Analyzes the conversation for key moments
2. Generates a summary
3. Logs the call to the contact record
4. Updates deal stage if there's explicit evidence
5. Creates follow-up tasks for commitments made
6. Updates analytics and metrics

</Accordion>

<Accordion title="Can I edit what Computer logged?">

Yes. Call records are fully editable:

1. Go to the contact's timeline
2. Find the call record
3. Click **Edit** to modify notes or outcome
4. Update next steps or add manual notes

</Accordion>

<Accordion title="Will Computer change my deal stages automatically?">

Computer only advances stages when there's explicit evidence:

| Commitment               | Stage Change               |
| ------------------------ | -------------------------- |
| "Let's schedule a demo"  | ✅ Moves to Demo Scheduled |
| "I'll send the proposal" | ✅ Moves to Proposal Sent  |
| "We're interested"       | ❌ Too vague               |
| "Seemed positive"        | ❌ Not a commitment        |

You can always manually adjust stages if Computer missed something important.

</Accordion>

<Accordion title="What if Computer creates duplicate tasks?">

Computer checks for existing similar tasks before creating new ones. If a similar task exists (same contact, similar title, pending), it updates the existing task instead of creating a duplicate.

</Accordion>

</AccordionGroup>

## CRM Tools

<AccordionGroup>

<Accordion title="What can Computer do in my CRM?">

Computer can:

- **Search** contacts, companies, deals
- **View** full records with history
- **Create** notes, tasks
- **Update** deal stages, amounts
- **Log** call outcomes
- **Queue** contacts for calling
- **Analyze** call metrics and trends

Computer cannot:

- Delete records
- Modify billing information
- Change workspace settings
- Access data your role can't see

</Accordion>

<Accordion title="How does Computer decide what to do?">

When you ask Computer to do something:

1. It parses your natural language request
2. Identifies which tool(s) can satisfy the request
3. Asks clarifying questions if parameters are missing
4. Executes the tool
5. Presents results in natural language

Example: "What deals are closing this month?"

1. Intent: List deals with specific date criteria
2. Tool: `list_deals` with date filter
3. Execute: Query CRM
4. Result: 5 deals found
5. Present: "You have 5 deals closing this month..."

</Accordion>

<Accordion title="Can I ask Computer to do something it's not sure about?">

Yes. Computer will:

- Ask clarifying questions if parameters are ambiguous
- Show you the action before taking it (for destructive operations)
- Explain what it will do and ask for confirmation
- Roll back changes if you disagree with the result

</Accordion>

</AccordionGroup>

## Privacy & Permissions

<AccordionGroup>

<Accordion title="What data does Computer access?">

Computer only accesses data your role permits:

- Contacts you can view
- Deals in your pipeline
- Notes and activity visible to you
- Knowledge base documents
- Integration data from connected tools

Hidden fields, disabled objects, and restricted records remain inaccessible.

</Accordion>

<Accordion title="Can Computer see my private notes?">

If a note is visible to your role, Computer can see it. If a note is private or restricted to specific users, Computer cannot access it — regardless of the role assigned.

</Accordion>

<Accordion title="Is my conversation stored?">

Call transcripts and coaching logs are stored in your workspace:

- Call recordings: Stored per your retention policy
- Transcripts: Stored with call records
- Coaching history: Stored in coaching session logs

Data stays in your workspace and is not used to train external models.

</Accordion>

<Accordion title="Can I control what Computer logs?">

Yes. You can:

- Edit call notes before or after logging
- Delete coaching sessions
- Configure which fields are visible to Computer
- Disable automatic logging in **Settings → Preferences**

</Accordion>

</AccordionGroup>

## Technical

<AccordionGroup>

<Accordion title="What AI model does Computer use?">

Computer uses OpenAI's GPT family optimized for:

- Real-time transcription latency
- Context understanding
- Natural language generation
- Structured output (JSON)

Model selection is configurable in workspace settings.

</Accordion>

<Accordion title="How does Computer protect my data?">

Security measures include:

- Encrypted transmission (TLS)
- Encrypted storage at rest
- Role-based access control
- Audit logging for all actions
- No data used for model training
- SOC 2 Type II compliant infrastructure

</Accordion>

<Accordion title="Can I use my own AI model?">

Enterprise customers can configure custom AI endpoints in **Settings → AI Configuration**. Self-hosted deployments support local models for air-gapped environments.

</Accordion>

<Accordion title="What integrations work with Computer?">

Computer integrates with:

- **Knowledge Base**: Upload PDFs, docs, resources
- **Integrations**: GoHighLevel, Salesforce, HubSpot (via API)
- **Data Enrichment**: Company and contact enrichment services

Contact sales for custom integration requests.

</Accordion>

</AccordionGroup>
