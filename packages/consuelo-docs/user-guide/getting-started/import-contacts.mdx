Consuelo allows you to import contacts via CSV files. This guide covers the import process, required fields, and best practices.

## Preparing Your CSV File

### Required Fields

| Field       | Description          | Format                            |
| ----------- | -------------------- | --------------------------------- |
| `firstName` | Contact's first name | Text                              |
| `lastName`  | Contact's last name  | Text                              |
| `phone`     | Phone number         | E.164 format (e.g., +14155551234) |

### Optional Fields

| Field     | Description          |
| --------- | -------------------- |
| `email`   | Email address        |
| `company` | Company name         |
| `title`   | Job title            |
| `notes`   | Any additional notes |

## Import Steps

1. Navigate to **Contacts** in the sidebar
2. Click the **Import** button
3. Select your CSV file
4. Map CSV columns to Consuelo fields
5. Review and confirm the import
6. Wait for the import to complete

## Phone Number Normalization

Consuelo automatically normalizes phone numbers to E.164 format during import. This ensures:

- Consistent formatting across all contacts
- Proper area code detection for local presence
- Accurate call routing

<Warning>
  If a phone number is invalid or cannot be normalized, the contact will be
  imported with a warning. Invalid numbers are marked in red.
</Warning>

## Handling Duplicates

During import, Consuelo checks for existing contacts with the same phone number or email. You can choose to:

- **Skip** - Don't import duplicates
- **Update** - Update existing contact information
- **Import All** - Import all contacts regardless of duplicates

## Troubleshooting

### Import Fails

- Ensure your CSV is properly formatted
- Check that required fields are present
- Verify phone numbers are in a recognized format

### Contacts Not Appearing

- Check if the import is still processing
- Verify field mapping was correct
- Ensure you have proper permissions
