Importing contacts allows you to quickly populate your CRM with existing data from spreadsheets or other systems.

## Supported File Formats

- CSV (Comma-Separated Values)
- Files must use UTF-8 encoding

## CSV Requirements

### Required Fields

| Field       | Description          | Example      |
| ----------- | -------------------- | ------------ |
| `firstName` | First name           | John         |
| `lastName`  | Last name            | Smith        |
| `phone`     | Phone number (E.164) | +14155551234 |

### Optional Fields

| Field     | Description      |
| --------- | ---------------- |
| `email`   | Email address    |
| `company` | Company name     |
| `title`   | Job title        |
| `address` | Full address     |
| `city`    | City             |
| `state`   | State/Province   |
| `zip`     | ZIP/Postal code  |
| `country` | Country          |
| `notes`   | Additional notes |

## Import Process

### Step 1: Prepare Your CSV

1. Export your data to CSV format
2. Ensure required columns are present
3. Format phone numbers to E.164 (include country code)

### Step 2: Start Import

1. Go to **Contacts**
2. Click **Import** in the top right
3. Click **Choose File** and select your CSV

### Step 3: Map Fields

1. The system auto-matches common field names
2. Map any unmatched columns manually
3. Preview the first few rows

### Step 4: Handle Duplicates

Choose how to handle existing contacts with matching data:

| Option     | Behavior                                     |
| ---------- | -------------------------------------------- |
| Skip       | Don't import contacts that already exist     |
| Update     | Update existing contacts with new data       |
| Import All | Import all contacts regardless of duplicates |

### Step 5: Import

1. Click **Start Import**
2. Watch the progress indicator
3. Review the summary when complete

## Phone Number Handling

Consuelo normalizes all phone numbers to E.164 format during import:

- `4155551234` → `+14155551234`
- `(415) 555-1234` → `+14155551234`

This ensures consistent formatting and enables local presence features.

## Troubleshooting

### Import Failed

- Check CSV formatting
- Verify required fields are present
- Ensure UTF-8 encoding

### Wrong Field Mapping

- Review column headers in your CSV
- Remap fields in step 3

### Duplicate Errors

- Choose a different duplicate handling strategy
- Clean up your data before importing

<Warning>
  Large imports (over 10,000 contacts) may take several minutes. Don't close the
  browser window until complete.
</Warning>
