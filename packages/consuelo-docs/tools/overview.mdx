# Tools

Tools are the callable capabilities agents use to do real work in Consuelo OS.

Use this section when you want an agent to call a named capability, pass typed input, receive structured output, and verify what changed. Tools are the operator-facing layer between agent instructions and OS runtime behavior.

## How to read tool docs

Each tool page uses the same shape:

| Section | What it gives you |
| --- | --- |
| When to use it | The job this tool performs |
| Mental model | What the tool consumes and produces |
| Recommended call | The facade call agents should prefer |
| CLI fallback | The shell command to use when the facade is unavailable |
| Validation | How to prove the tool worked |
| Common failures | What to check when the tool returns an error |

## Steering rule for agents

Give agents the exact tool name, explicit input and output paths, JSON mode, and a validation requirement. The best instruction says what tool to call and what file or artifact must exist when the call finishes.

```text
Use the existing OS tool. Do not implement a new converter, downloader, renderer, or wrapper.
Call the named tool with the provided input.
Return the JSON result.
Verify the expected output file or manifest exists.
Report the tool result and any fallback behavior.
```
