# Media Capabilities

Media tools are grouped by the job they perform. Start with the workflow, then choose the tool that owns the next step.

## Image and SVG

| Tool | Use it for |
| --- | --- |
| [`media.svg.convert`](/tools/media/reference/media-svg-convert) | Convert PNG or JPEG images into SVG wrapper files, traced vector SVGs, or both. |

Use this group for generated images, brand assets, simple logos, stickers, icons, and media-kit graphics.

## Source capture and ingest

| Tool | Use it for |
| --- | --- |
| [`media.ingest`](/tools/media/reference/media-ingest) | Turn a local file or media URL into a source media asset bundle. |

Ingest is the boundary that preserves source material. It should create asset metadata, source info, checksums, and an ingest manifest.

## Video inspection

| Tool | Use it for |
| --- | --- |
| [`media.probe`](/tools/media/reference/media-probe) | Inspect a source media file with `ffprobe`. |
| [`media.frames.extract`](/tools/media/reference/media-frames-extract) | Extract frames for review, thumbnails, overlays, and analysis. |

Use these tools before writing a timeline or making claims about a video. They establish duration, dimensions, streams, and visual evidence.

## Timeline and rendering

| Tool | Use it for |
| --- | --- |
| [`media.timeline.validate`](/tools/media/reference/media-timeline-validate) | Check timeline structure and referenced assets before rendering. |
| [`media.compose`](/tools/media/reference/media-compose) | Render a timeline into a draft video. |
| [`media.overlay.render`](/tools/media/reference/media-overlay-render) | Render declarative overlays for composition. |

Timelines are the edit decision layer. They describe the asset, output format, beats, captions, overlays, and render settings.

## QA and export

| Tool | Use it for |
| --- | --- |
| [`media.qa`](/tools/media/reference/media-qa) | Inspect rendered output for basic quality and contract checks. |
| [`media.export`](/tools/media/reference/media-export) | Package rendered media for Shorts, TikTok, Reels, longform YouTube, and handoff workflows. |

QA and export turn a render into something another person or system can review.

## Audio and captions

| Tool | Use it for |
| --- | --- |
| [`media.transcribe`](/tools/media/reference/media-transcribe) | Create transcript artifacts from media audio or video. |

Transcription supports captions, notes, search, and edit decisions. Model downloads should stay explicit.

## Vision and sports analysis

| Tool | Use it for |
| --- | --- |
| [`media.scene.detect`](/tools/media/reference/media-scene-detect) | Find scene boundaries and candidate highlight moments. |
| [`media.motion.track`](/tools/media/reference/media-motion-track) | Track generic motion vectors and visual features. |
| [`media.pose.estimate`](/tools/media/reference/media-pose-estimate) | Estimate body landmarks for pose analysis. |
| [`media.breakdown.plan`](/tools/media/reference/media-breakdown-plan) | Build a structured, data-backed explanation plan. |

Use these tools when a media output needs evidence-backed analysis. Claims should point to timestamps, frames, pose tracks, motion tracks, or measured metrics.

## Dependency profiles

| Profile | Enables |
| --- | --- |
| `media-core` | Probe, frame extraction, timeline validation, compose, QA, export |
| `media-youtube` | URL ingest and clip discovery |
| `media-audio` | Audio extraction, normalization, and transcription support |
| `media-vision-light` | Scene detection and generic motion tracking |
| `media-vision-pose` | Human pose landmarks and sports-science metrics |
| `media-render-advanced` | SVG tracing, overlays, advanced render assets |

Run `media.doctor` or the install-plan tools when a capability returns a missing dependency error.
