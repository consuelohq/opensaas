# Image to SVG

This workflow converts a generated image, logo, sticker, or simple graphic into SVG files through `media.svg.convert`.

Use it when you need both fidelity and editability. The wrapper SVG preserves the original raster image. The traced SVG gives you vector paths that can be edited in design tools.

## Workflow

<Steps>
  <Step title="Prepare the image">
    Save the source as a PNG or JPEG and choose the requested SVG output path.
  </Step>
  <Step title="Run media.svg.convert">
    Use `strategy: both` so the tool writes the traced SVG to the requested output path and the wrapper SVG beside it.
  </Step>
  <Step title="Review both outputs">
    Use the traced SVG when editability matters. Use the wrapper SVG when exact visual preservation matters.
  </Step>
</Steps>

## Recommended call

```json
{
  "input": "assets/generated-logo.png",
  "out": "artifacts/generated-logo.svg",
  "strategy": "both",
  "traceEngine": "color",
  "optimize": false
}
```

## CLI fallback

```bash
bun run --cwd packages/os media -- svg convert   --input assets/generated-logo.png   --out artifacts/generated-logo.svg   --strategy both   --trace-engine color   --json
```

## Agent instruction

```text
Use the existing OS tool `media.svg.convert`.
Input: assets/generated-logo.png
Output: artifacts/generated-logo.svg
Strategy: both
Trace engine: color if `vtracer` is available, otherwise auto.
Return JSON.
Verify artifacts/generated-logo.svg exists.
Report outputs.svg, outputs.wrapperSvg, outputs.tracedSvg, traceEngine, and optimized.
Do not write a custom SVG converter.
```

## Validation

Check these facts after the tool returns:

| Check | Expected result |
| --- | --- |
| Requested output path | Exists and points to the traced SVG |
| Wrapper output | Exists at the derived `.wrapper.svg` path |
| JSON schema | `media.svg-result.v1` |
| Trace engine | `color`, `mono`, or `auto` fallback reported |

## Common failures

| Failure | Fix |
| --- | --- |
| `vtracer` missing | Install it with `cargo install vtracer`, then make sure `$HOME/.cargo/bin` is on PATH. |
| `svgo` missing | Install it with `brew install svgo` before using `optimize: true`. |
| Unsupported input | Use PNG or JPEG. |
| Output missing | Check the returned `outputs` object and the requested `out` path. |
