# YouTube Clip Breakdown

This workflow turns a candidate YouTube clip into a source asset, review frames, analysis artifacts, a breakdown plan, and a renderable timeline.

Use it when the output needs an explanation grounded in timestamps, frames, motion, pose, or measured facts.

## Workflow

<Steps>
  <Step title="Find a candidate clip">
    Use `media.clip.search` or a provided URL to select one source clip.
  </Step>
  <Step title="Ingest the source">
    Run `media.ingest` so the workflow preserves actual source media and provenance.
  </Step>
  <Step title="Inspect and sample">
    Run `media.probe` and `media.frames.extract` to establish duration, dimensions, and review frames.
  </Step>
  <Step title="Analyze the moment">
    Use `media.scene.detect`, `media.motion.track`, or `media.pose.estimate` when the claim needs visual evidence.
  </Step>
  <Step title="Plan the breakdown">
    Run `media.breakdown.plan` to connect claims to timestamps, frames, and analysis artifacts.
  </Step>
  <Step title="Render and export">
    Validate the timeline, compose the video, run QA, and package the export.
  </Step>
</Steps>

## Source rule

`media.ingest` must preserve real source media. Transcript text, screenshots, and notes are useful sidecars. They are not the source asset.

## Agent instruction

```text
Build a YouTube clip breakdown with the existing media tools.
Use `media.ingest` for source capture.
Preserve the actual source media file.
Run probe and frame extraction before making claims.
Use motion or pose analysis only when the claim needs it.
Create a breakdown plan that ties every claim to timestamps, frames, metrics, or source provenance.
Return JSON from each tool and verify every referenced file exists.
```

## Claim discipline

Use this rule for analysis prose:

| Claim type | Required evidence |
| --- | --- |
| Visual observation | Timestamp or frame reference |
| Motion claim | Motion-track or frame-to-frame evidence |
| Body mechanics claim | Pose landmarks or angle measurements |
| Coaching takeaway | Breakdown plan with source references |

## Common failures

| Failure | Fix |
| --- | --- |
| URL capture fails | Read the structured `yt-dlp` error and retry only when the error is transient. |
| Source file missing | Treat the ingest as failed. The source media asset is required. |
| Claim has no evidence | Add frame, timestamp, motion, pose, or metric evidence before rendering. |
| Rights status is unresolved | Keep `rights.status = needs-review` for review workflows. |
