# media.export

Package rendered media for platform targets and handoff workflows.

## When to use it

Use this after QA passes and the render needs a platform-specific package or artifact handoff.

## Mental model

Export is packaging. It keeps the rendered video, thumbnail, captions, notes, provenance, rights status, and target metadata together.

## Inputs

| Field | Meaning |
| --- | --- |
| `renderResult` | Path to `media.render-result.v1`. |
| `target` | Export target such as `youtube-shorts`, `tiktok`, `reels`, or `longform-youtube`. |
| `outDir` | Directory for the export package. |

## Output

Returns `media.export-package.v1` with files, target, provenance, rights status, and deterministic package metadata.

## CLI fallback

```bash
bun run --cwd packages/os media -- export --render-result artifacts/media/render-result.json --target youtube-shorts --out artifacts/media/export --json
```

## Good agent instruction

```text
Use `media.export` after QA passes.
Return JSON.
Verify the package references an MP4, target, source asset, and rights status.
```

## Validation

| Check | Expected result |
| --- | --- |
| MP4 reference | The package references an MP4 file. |
| Target | The package has a platform target. |
| Provenance | Source asset and rights status remain attached. |
