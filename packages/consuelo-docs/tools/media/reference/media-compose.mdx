# media.compose

Render a valid media timeline into a draft video.

## When to use it

Use this after timeline validation passes and the workflow needs a rendered output file.

## Mental model

Compose consumes a timeline and writes a video plus a render result manifest. It is the deterministic render step in the media loop.

## Inputs

| Field | Meaning |
| --- | --- |
| `timeline` | Path to a validated `media.timeline.v1` file. |
| `out` | Path for the rendered video. |

## Output

Returns `media.render-result.v1` with output path, render metadata, source references, and tool evidence.

## CLI fallback

```bash
bun run --cwd packages/os media -- compose --timeline artifacts/media/timeline.json --out artifacts/media/render.mp4 --json
```

## Good agent instruction

```text
Use `media.compose` only after timeline validation passes.
Return JSON.
Verify the rendered output path exists.
Run `media.qa` next.
```

## Validation

| Check | Expected result |
| --- | --- |
| Rendered file | The video exists at the requested output path. |
| Result schema | The result uses `media.render-result.v1`. |
| Next gate | `media.qa` can inspect the rendered file. |
