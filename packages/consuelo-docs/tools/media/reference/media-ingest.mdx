# media.ingest

Capture a local file or media URL into a deterministic media asset bundle.

## When to use it

Use this tool at the beginning of a video workflow when source media needs metadata, checksums, provenance, and a preserved source file.

## Mental model

Ingest owns source capture. It writes a bundle with source media, source info, asset metadata, and an ingest manifest. Sidecar text can help later tools. The actual media file is the required source asset.

## Inputs

| Field | Meaning |
| --- | --- |
| `source` | Local media path or supported URL. |
| `outDir` | Directory where the ingest bundle should be written. |
| `dryRun` | Plan capture without writing source media when set. |

## Output

Returns `media.ingest-manifest.v1` with asset references, source info, byte sizes, checksums, and rights status.

## Recommended call

```json
{
  "source": "assets/source.mp4",
  "outDir": "artifacts/media/source-001",
  "dryRun": false
}
```

## CLI fallback

```bash
bun run --cwd packages/os media -- ingest \
  --source assets/source.mp4 \
  --out artifacts/media/source-001 \
  --json
```

## Good agent instruction

```text
Use the existing OS tool `media.ingest`.
Preserve the actual source media file.
Return JSON.
Verify the ingest manifest and source media file exist.
Keep rights status as needs-review unless a human has reviewed it.
```

## Validation

| Check | Expected result |
| --- | --- |
| Source media exists | The bundle contains the preserved source file. |
| Manifest exists | The ingest manifest references the source asset. |
| Rights status | Default status remains `needs-review` until reviewed. |

## Common failures

| Failure | Fix |
| --- | --- |
| Source file missing | Treat ingest as failed. |
| URL capture error | Read the structured capture error and retry only when transient. |
| Storage budget exceeded | Use a shorter source or explicit budget decision. |
