# media.timeline.validate

Validate a media timeline before rendering.

## When to use it

Use this before `media.compose` to catch missing assets, invalid timings, unsupported settings, and malformed references.

## Mental model

A timeline is the edit decision list. Validation proves the plan is safe to render.

## Inputs

| Field | Meaning |
| --- | --- |
| `timeline` | Path to a `media.timeline.v1` file. |

## Output

Returns a timeline validation result with pass or fail status and validation issues.

## CLI fallback

```bash
bun run --cwd packages/os media -- timeline validate --timeline artifacts/media/timeline.json --json
```

## Good agent instruction

```text
Use `media.timeline.validate` before rendering.
Return JSON.
Fix blocking issues before calling `media.compose`.
```

## Validation

| Check | Expected result |
| --- | --- |
| Status | No blocking issues. |
| References | Assets in the timeline resolve to real paths. |
| Timing | Beats and overlays stay inside source and output duration. |
