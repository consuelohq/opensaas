# media.qa

Inspect rendered media for duration, dimensions, codec, size, captions, and quality checks.

## When to use it

Use this immediately after `media.compose` and before `media.export`.

## Mental model

QA checks the rendered output against expected media facts. It is the quality gate before packaging and handoff.

## Inputs

| Field | Meaning |
| --- | --- |
| `input` | Path to rendered media. |
| `expect` | Optional duration, dimension, or package constraints. |

## Output

Returns `media.qa-result.v1` with pass or fail status, media facts, and issues.

## CLI fallback

```bash
bun run --cwd packages/os media -- qa --input artifacts/media/render.mp4 --json
```

## Good agent instruction

```text
Use `media.qa` after rendering.
Return JSON.
Report blocking QA issues before exporting.
```

## Validation

| Check | Expected result |
| --- | --- |
| QA status | No blocking quality issues. |
| Media facts | Duration, dimensions, and codecs are included. |
| Output path | The checked media file exists. |
