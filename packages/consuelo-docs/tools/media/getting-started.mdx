# Media Tools

Media tools let agents work with images, SVGs, video, audio, overlays, transcripts, and rendered export packages through named OS capabilities.

Use media tools when the task needs deterministic media work: converting an image, ingesting source footage, inspecting a video, extracting frames, authoring a timeline, rendering a draft, checking output quality, or packaging a final export.

## The media loop

The core workflow is:

```text
source asset
  -> inspect
  -> transform or analyze
  -> timeline
  -> compose
  -> QA
  -> export
  -> artifact handoff
```

Agents should move through that loop instead of improvising one-off scripts. Each step returns structured JSON and writes predictable files.

## First thing to try

Convert an image to SVG with `media.svg.convert`.

```json
{
  "input": "assets/logo.png",
  "out": "artifacts/logo.svg",
  "strategy": "both",
  "traceEngine": "auto",
  "optimize": false
}
```

This produces a traced SVG at the requested output path and a wrapper SVG beside it. The traced SVG is editable vector output. The wrapper SVG preserves the original image exactly by embedding the raster source.

## Recommended agent instruction

Use this shape when asking an agent to work with media:

```text
Use the existing Consuelo OS media tool for this job.
Tool: <tool-name>
Input: <source path or manifest>
Output: <expected path>
Return JSON.
Verify the expected file or manifest exists.
Do not write a custom converter, downloader, renderer, or analysis script unless the tool itself fails and the failure is recorded.
```

## What media tools protect

Media tools protect the workflow from three common failures:

1. **Untracked source material.** Ingest records where media came from, where it was written, and which asset was preserved.
2. **Unverifiable output.** Render and export tools return manifests so later agents can check paths, dimensions, duration, and provenance.
3. **Agent drift.** Tool docs give agents exact tool names, input shapes, commands, and validation steps.
