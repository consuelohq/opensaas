## Overview

Integrate Consuelo with Slack to notify your team in real-time when important events happen. Use webhooks to post messages to channels when deals close, leads come in, or workflows complete.

## Setting Up Slack Webhooks

### Step 1: Create a Slack App

1. Go to [api.slack.com/apps](https://api.slack.com/apps)
2. Click **Create New App**
3. Choose **From scratch**
4. Name it "Consuelo Notifications"
5. Select your workspace

### Step 2: Enable Incoming Webhooks

1. In your app settings, go to **Incoming Webhooks**
2. Toggle **Activate Incoming Webhooks** to On
3. Click **Add New Webhook to Workspace**
4. Select the channel to post to (e.g., #sales-alerts)
5. Click **Allow**
6. Copy the webhook URL

### Step 3: Store Webhook URL

Save the URL for use in Consuelo:

```
<paste-your-slack-webhook-url-here>
```

<Warning>
  **Keep this URL secret.** Anyone with this URL can post to your Slack channel.
</Warning>

## Creating Slack Notification Workflows

### Basic Notification

**Trigger:** Record created (Person)
**Actions:**

1. **HTTP Request:** POST to Slack webhook

**Configuration:**

```
Method: POST
URL: <paste-your-slack-webhook-url-here>
Headers:
  Content-Type: application/json
Body: {"text": "New lead: {{trigger.object.firstName}} {{trigger.object.lastName}}"}
```

## Common Notification Patterns

### Lead Alert

Notify sales team of new qualified leads:

**Trigger:** Record created (Person)
**Filter:** Lead score > 50
**Actions:**

1. **HTTP Request:** POST to Slack #sales-alerts
2. **Create Record:** Task for sales rep

### Deal Won Celebration

Celebrate closed deals:

**Trigger:** Opportunity updated (stage = "Closed Won")
**Actions:**

1. **HTTP Request:** POST to Slack #wins
2. **Send Email:** Notify stakeholders

### Daily Digest

Send daily summary:

**Trigger:** Daily 9am
**Actions:**

1. **Search Records:** Opportunities updated yesterday
2. **Code:** Format summary
3. **HTTP Request:** POST to Slack #daily-updates

### Data Quality Alert

Notify of data issues:

**Trigger:** Workflow run fails
**Filter:** Error count > 5
**Actions:**

1. **HTTP Request:** POST to Slack #data-alerts
2. **Create Record:** Task for data steward

### Dynamic Content

Use Code action to generate message content:

```javascript
// Format deal value
const amount = trigger.object.amount;
const formatted = new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD',
}).format(amount);

return {
  message: `💰 *Deal Won!*\n\n{{trigger.object.name}} - ${formatted}`,
  value: formatted,
};
```

## Rate Limiting

Slack has rate limits for webhooks:

- **General:** 1 message per second
- **Burst:** Up to 5 messages per second briefly

**Best practices:**

- Add delays between notifications
- Batch notifications when possible
- Use scheduled digests instead of individual alerts

**Example: Adding delays**

```
Send notification 1
↓
Delay 2 seconds
↓
Send notification 2
↓
Delay 2 seconds
↓
Send notification 3
```

## Error Handling

### Handling Failures

Webhook calls may fail. Handle gracefully:

1. **HTTP Request:** POST to Slack
2. **Filter:** Check if response contains "ok"
3. **Branch:**
   - **Success:** Continue
   - **Failure:** Send email alert

### Retry Logic

Consuelo workflows don't have built-in retry. Work around this:

1. **HTTP Request:** First attempt
2. **Filter:** Check success
3. **If failed:**
   - **Delay:** Wait 30 seconds
   - **HTTP Request:** Retry
4. **If still failed:**
   - **Send Email:** Alert admin

## Security

### Protecting Webhook URLs

**Never hardcode URLs in workflows.** Use these methods:

1. **Environment variables** (if available in your deployment)
2. **Reference records** (store in Consuelo, query in workflow)
3. **Code action** (fetch from secure storage)

**Example: Using a reference record**

1. Create a "Config" object with fields:
   - Name: "Slack Webhook"
   - Value: "https://hooks.slack.com/..."

2. In workflow:
   - **Search Records:** Config where name = "Slack Webhook"
   - **HTTP Request:** Use `{{searchRecords.webhookUrl}}`

### Testing in Private Channels

Before posting to public channels:

1. Create a private test channel
2. Add webhook to test channel
3. Test workflow thoroughly
4. Switch to production channel

## Troubleshooting

### Messages Not Appearing

| Issue              | Solution                     |
| ------------------ | ---------------------------- |
| Webhook URL wrong  | Verify URL matches Slack app |
| JSON malformed     | Validate JSON syntax         |
| Rate limited       | Check Slack webhook logs     |
| Channel archived   | Unarchive or change channel  |
| App not in channel | Invite app to channel        |

### Formatting Issues

| Issue                | Solution                         |
| -------------------- | -------------------------------- |
| Emojis not showing   | Use Unicode or Slack emoji names |
| Links not clickable  | Include full URL with https://   |
| Text truncated       | Keep under 4000 characters       |
| Blocks not rendering | Use Block Kit builder to test    |

## Best Practices

### Message Content

- **Be concise:** Under 200 words
- **Include links:** Deep link to Consuelo records
- **Add context:** Why does this matter?
- **Use emojis:** Visual hierarchy
- **Format numbers:** Currency, percentages

### Channel Strategy

| Channel        | Purpose                          |
| -------------- | -------------------------------- |
| #sales-alerts  | Real-time lead notifications     |
| #wins          | Closed deals, milestones         |
| #data-alerts   | Data quality issues              |
| #daily-updates | Scheduled summaries              |
| #errors        | Workflow failures, system alerts |

### Timing

- **Avoid spam:** Don't notify on every update
- **Use filters:** Only notify on important changes
- **Batch when possible:** Daily digest vs individual alerts
- **Respect hours:** Don't send alerts at 3am
