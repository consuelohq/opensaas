## implementation notes

### bug: garbled sidebar labels (2026-03-05)
- Dialer showed "e r t d k y", Chat showed "ah z f l p" — these are Lingui hash IDs (ErTdky, AHZflp) displayed with spaces between characters
- Root cause: `t`Dialer`` and `t`Chat`` were new strings not in the Lingui .po catalogs. The `@lingui/swc-plugin` generates hash IDs at compile time. When the compiled catalog doesn't have the entry, Lingui displays the hash ID instead of falling back to the source string.
- Fix: Added "Dialer" and "Chat" msgid/msgstr entries to all 32 .po locale files, then ran `lingui compile --typescript` to regenerate the compiled catalogs.
- "Agent", "Skills", "Workspace" already existed in the .po files from prior twenty codebase usage.
- Note: `npx nx lingui:extract twenty-front` is broken due to a pre-existing babel parser error in `settingsAccountsSelectedMessageChannelState.ts` (duplicate `createState` identifier). New Lingui strings must be manually added to .po files until this is fixed.
