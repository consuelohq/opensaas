# workpad: DEV-1074 — [task] 13.4: Phone Number Provisioning UI
# created: 2026-03-04T01:43:11Z

## acceptance criteria

- [x] hosted user can search available numbers by area code
- [x] hosted user can provision a number (purchased on sub-account)
- [x] BYOK user can see their existing numbers AND buy new ones
- [x] first number is auto-set as primary
- [x] number immediately usable for outbound calls after provisioning
- [x] can release a number from the UI

## implementation notes

### approach
- dialer package: add AvailableNumber type, SearchAvailableNumbersOptions, ReleaseResult to types.ts
- base provider: add searchAvailableNumbers() and releaseNumber() to interface
- twilio provider: implement both methods (availablePhoneNumbers API + incomingPhoneNumbers remove)
- dialer class: delegation methods
- backend: 4 new routes in voice.ts (available, provision, :sid DELETE, :sid/primary PUT)
- redis: setPrimaryNumber/getPrimaryNumber for workspace-scoped primary storage
- frontend: 4 hooks + AddPhoneNumberModal + update PhoneNumberSettings
- primary auto-set: provision route checks if first number, auto-sets primary
- ROUTE_ORDER: literal routes (available, provision) before param routes (:sid, :sid/primary)

### key decisions
- use Redis for primary number storage (key: primary-number:{workspaceId})
- follow TransferModal overlay pattern for AddPhoneNumberModal (not Twenty's Modal system)
- all routes use getDialerForWorkspace() for multi-tenant support
- provision route sets voiceUrl to TwiML webhook automatically

## improvements noticed

- dialer package tsconfig.json doesn't have `declaration: true` — relies on base tsconfig which has `declaration: false`. the dist/*.d.ts files were generated with `--declaration` flag manually. should add `"declaration": true` to dialer tsconfig.json so `npx tsc` generates declarations by default.
