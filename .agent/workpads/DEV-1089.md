## implementation notes

### root cause
the GHLSettings.tsx page had TWO bugs causing the 401:
1. **wrong path/method**: page called `GET /v1/integrations/ghl/auth`, hook expected `POST /v1/integrations/ghl/oauth`
2. **missing auth header**: page's `fetchJson` used `credentials: 'include'` (cookies) but backend's `requireAuth` checks `Authorization: Bearer` header

### fix applied
1. **backend** (`packages/api/src/routes/ghl.ts`): renamed `GET /auth` → `POST /oauth`, changed response from `{ url, state }` to `{ redirectUrl: url, state }`
2. **frontend** (`packages/twenty-front/src/pages/settings/consuelo/GHLSettings.tsx`):
   - imported `getTokenPair` from `~/apollo/utils/getTokenPair`
   - updated `fetchJson` to send `Authorization: Bearer` header
   - updated `handleConnect` to call `POST /oauth` and expect `redirectUrl`

### alignment
all three paths now match: backend POST /oauth, page calls /oauth, hook calls /oauth
