## root cause analysis

### 4 bugs found:

1. **generateFrontConfig() regex mismatch** — the function looks for `<!-- BEGIN: Twenty Config -->` but the HTML was rebranded to `<!-- BEGIN: Consuelo Config -->`. the regex never matches, so `window._env_` is never injected in production. REACT_APP_SERVER_BASE_URL falls through to getDefaultUrl() which returns the current origin (correct fallback).

2. **SERVER_URL env var wrong on Railway** — was set to `https://consuelohq.com` (bare domain, redirects to www). should be `https://consuelo.consuelohq.com` (the actual app URL). if the regex WERE fixed without fixing this, the frontend would make cross-origin requests to the wrong domain.

3. **CORS config incompatible with credentials** — `cors: true` sets `Access-Control-Allow-Origin: *` with no `Access-Control-Allow-Credentials: true`. the frontend uses `credentials: 'include'` in authenticatedFetch. for cross-origin requests, the browser requires a specific origin (not *) and credentials:true. confirmed via curl: preflight returns `access-control-allow-origin: *` with no credentials header.

4. **health route missing auth:false** — the /health endpoint was behind auth middleware, returning 401 for unauthenticated health checks.

### the 401 chain:
- generateFrontConfig regex doesn't match → window._env_ not set → REACT_APP_SERVER_BASE_URL falls to getDefaultUrl() → same-origin requests work
- BUT: CORS config with `origin: *` + no credentials support means any cross-origin scenario fails silently
- the fix: update regex + fix SERVER_URL + fix CORS + fix health route

## fixes applied

1. `packages/twenty-server/src/utils/generate-front-config.ts` — regex now matches both "Twenty Config" and "Consuelo Config", replacement text uses "Consuelo Config"
2. `packages/twenty-server/src/main.ts` — CORS changed from `cors: true` to `cors: { origin: true, credentials: true }` — reflects request origin and supports credentials
3. `packages/api/src/routes/index.ts` — health route now has `auth: false`
4. Railway: `SERVER_URL` updated from `https://consuelohq.com` to `https://consuelo.consuelohq.com`
